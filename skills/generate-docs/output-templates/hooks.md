Example walkthrough for `useCreateInspection`:

This is a TanStack Query mutation that wraps the `POST /api/v1/inspections/` call. On submit, it optimistically inserts the new inspection into the `['inspections']` cache so the list updates instantly. If the request succeeds, the cache is replaced with the server's authoritative version (canonical IDs, server timestamps). If it fails, the optimistic insert is rolled back and the error surfaces via the `onError` handler â€” the calling component shows a toast.

After success, the hook invalidates `['inspections']`, `['inspection-counts']`, and `['client', clientId, 'inspections']` so any nearby query that depends on the inspection list refetches automatically. The user-perceived latency is just the optimistic insert; everything else happens in the background.

</details>
